# Campercontact Moderator Tools

Flask frontend for the Campercontact AI agent pipeline. Moderators can enter one or more profile IDs, trigger the full backend pipeline, and review the final Confidence Agent output.

## Current backend flow

```text
Gap Detector → Source Finder → Entity Matcher → Hint Generator → Confidence Agent → BigQuery
```

The frontend treats this table as the final source of truth:

```text
project-62cd3637-0b98-4aa5-8d5.primary_dataset.hint_confidence_results
```

The anomaly/priority queue page reads from:

```text
project-62cd3637-0b98-4aa5-8d5.hint_prioritization.moderator_hint_queue_with_anomaly_v1
```

## What the frontend calls

The browser only calls the Flask app. Flask then calls the private Gap Detector Cloud Run service:

```text
POST {GD_URL}/run-gap-detector
```

Payload generated by Flask for one profile is equivalent to:

```json
{
  "profile_ids": ["19060"],
  "trigger_source_finder": true,
  "source_finder_limit": 1,
  "trigger_entity_matcher": true,
  "entity_matcher_limit": 5,
  "trigger_hint_generator": true,
  "hint_generator_limit": 5,
  "trigger_confidence_agent": true,
  "confidence_agent_limit": 5,
  "refresh_prioritization": true,
  "write_bigquery": true
}
```

The results page then polls Flask, and Flask reads BigQuery rows from `hint_confidence_results` filtered by `gap_detector_run_id` and `profile_id`.

## Cloud Run deployment

From this folder:

```bash
gcloud run deploy campercontact-frontend \
  --source . \
  --region=europe-west1 \
  --service-account=campercontact-frontend-sa@project-62cd3637-0b98-4aa5-8d5.iam.gserviceaccount.com \
  --set-env-vars BIGQUERY_PROJECT=project-62cd3637-0b98-4aa5-8d5,GD_URL=https://gap-detector-agent-tbq24k2wna-ew.a.run.app,GD_AUTH=true,POLLING_INTERVAL_SECONDS=5,POLLING_TIMEOUT_SECONDS=600,PIPELINE_REQUEST_TIMEOUT_SECONDS=300
```

The `Procfile` starts the app with Gunicorn on Cloud Run's `$PORT`.

## IAM needed

The frontend service account needs:

- Cloud Run Invoker on the private Gap Detector service.
- BigQuery Job User on the project.
- BigQuery read access to `primary_dataset.hint_confidence_results` and `hint_prioritization.moderator_hint_queue_with_anomaly_v1`.

## UI confidence logic

- `HIGH`: shown as recommended updates.
- `MEDIUM`: shown as review-needed hints.
- `LOW`: collapsed/hidden by default in a low-confidence section.

Technical fields such as source URL, reliability score, uplift score, contradiction penalty, `confidence_id`, and `hint_id` are hidden inside a collapsed technical details panel.

## Timestamp handling

BigQuery timestamps are treated as UTC. The frontend serializes them with a `Z` suffix and displays them in `Europe/Amsterdam` time with an explicit `Amsterdam time` label.
